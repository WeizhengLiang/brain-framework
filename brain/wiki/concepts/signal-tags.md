# Signal Tags

## Definition
Lightweight labels appended to completed tasks in `done.md` that indicate whether the task produced learning worth capturing. They are the tripwire for the brain's self-learning system — most tasks are `#routine`, and the non-routine tags trigger deeper analysis.

## Key Points
- Four values: `#routine`, `#had-friction`, `#surprised`, `#failed-first`
- `#routine` means nothing to learn — move on. The other three trigger a Tier 2 retrospective via `/review`
- Tags also appear in git commit messages for task commits, making `git log` a learning signal
- Choosing the right tag is a judgment call by the Orchestrator at `/done` time
